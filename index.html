Sistema de Visión para
Detección de Defectos
Proyecto académico en colaboración con KIA — Tec de Monterrey, 2024–2025
+

Desarrollé un sistema de visión artificial end-to-end para detectar fallas en blanks metálicos dentro de una línea de manufactura. El reto era identificar defectos de forma autónoma, en tiempo real, con la precisión necesaria para operar en un entorno industrial real.

El Problema
  • La inspección visual de blanks metálicos se realizaba manualmente, generando inconsistencias y fatiga del operador.
  • Los defectos no detectados a tiempo generaban desperdicios costosos en etapas posteriores del proceso.
  • No existía registro histórico automatizado de los defectos para análisis de calidad.
Solución Desarrollada
1
Captura de Datos
Configuración de cámara industrial para la captura de imágenes de blanks metálicos en condiciones de línea de producción. Definición del pipeline de adquisición de datos.
2
Preprocesamiento e Ingeniería de Datos
Limpieza, etiquetado y augmentación del dataset de imágenes. Normalización y estructuración para entrenamiento de modelos de detección de objetos.
3
Entrenamiento del Modelo YOLO
Entrenamiento y validación de modelo YOLO para clasificación y localización de defectos en imágenes. Iteraciones de ajuste para alcanzar alta precisión en detección.
4
Pipeline en Tiempo Real con Raspberry Pi
Integración del modelo en una Raspberry Pi para inferencia en el borde. El sistema procesa imágenes en tiempo real y envía los datos de detección al sistema de monitoreo.
5
Dashboard de Monitoreo en Grafana
Construcción de dashboards en Grafana con flujo de datos en tiempo real para visualizar defectos detectados, frecuencias y tendencias de calidad. El prototipo se configuró para escalabilidad en planta.

Proyecto para Komatsu México — Optimización del proceso de refacciones, 2024
+

Desarrollé un diagnóstico estratégico y una solución de automatización de datos para Komatsu México, empresa líder en maquinaria pesada para minería, construcción y sector forestal. El objetivo: intervenir un proceso de suministro de refacciones altamente fragmentado para sentar las bases de un sistema de tracking en tiempo real.

El Reto de Negocio
  • Un paro de operación por falta de una refacción representa pérdidas económicas críticas para los clientes industriales de Komatsu.
  • El incremento de aranceles con EE.UU. migró el suministro hacia Japón, extendiendo los tiempos de tránsito y complejizando la cadena de suministro global.
  • El tracking de refacciones dependía de llamadas, correos y hojas de cálculo — sin visibilidad centralizada ni autoservicio para el cliente.
  • La saturación reactiva del equipo interno impedía actividades de mayor valor estratégico.
Solución en Dos Etapas
1
Diagnóstico Estratégico y Benchmarking
Modelé el proceso logístico actual mediante Business Model Canvas y swimlane diagrams. Realicé un benchmarking competitivo contra Caterpillar (parts.cat.com + WMSv2) y John Deere evaluando 6 dimensiones: portal de autoservicio, tracking en tiempo real, ETA dinámico, integración telemática, inteligencia artificial y visibilidad al cliente final.
2
Motor de Clasificación de Compras en Python
Desarrollé un algoritmo en Python que procesa registros históricos de órdenes de compra del ERP de Komatsu. El sistema limpia, categoriza y subcategoriza automáticamente cada orden según tipo de componente (motor, hidráulicos, consumibles) y urgencia logística — creando la base estructurada necesaria para un sistema de notificaciones automáticas.
Metodologías Aplicadas
  • Business Model Canvas para análisis del modelo de negocio B2B de Komatsu.
  • Swimlane Diagrams para mapeo del flujo logístico AS-IS y detección de puntos de quiebre.
  • Benchmarking de procesos contra estándares de clase mundial en la industria.
  • Categorización inteligente de datos con Pandas y NLP básico para clasificación de texto.
